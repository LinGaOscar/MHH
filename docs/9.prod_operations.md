本文件詳述生產環境（PROD）之維運規範，涵蓋部署路徑、啟動程序、日常監控及故障排除流程。

## 1. PROD 環境部署規範

本節提供維運人員將系統部署至生產環境 Windows Server 之標準流程。

### 1.1 環境準備
- **Runtime**: Java 21 JRE
- **Database**: 獨立安裝之 MS SQL Server 2019+
- **權限需求**: 執行帳戶須對電文目錄（如 `D:\MHH_FILES`）具備完全控制權。

### 1.2 目錄結構規範 (建議)
```text
D:\MHH\
├── bin\           # 存放 mhh-ap.jar, mhh-batch.jar
├── config\        # 存放 application-prod.yml
├── logs\          # 系統產生的日誌檔
└── data\          # 電文處理目錄 (PDF 來源、歸檔、暫存)
```

### 1.3 打包與部署流程
1. **執行打包**：在開發環境或 CI 伺服器執行 `mvn clean package -DskipTests`。
2. **收集產出物**：
   - 網頁應用：`mhh-ap/target/mhh-ap.jar`
   - 排程任務：`mhh-batch/target/mhh-batch.jar`
   - 配置文件：從 `src/main/resources/application.yml` 複製並依環境修改。

### 1.4 PROD 啟動步驟
建議將 JAR 與 `application.yml` 分離存放，啟動指令如下：

**啟動 Web 應用：**
```powershell
java -jar mhh-ap.jar --spring.config.location=file:./config/application-prod.yml
```

**啟動 Batch 服務：**
```powershell
java -jar mhh-batch.jar --spring.config.location=file:./config/application-prod.yml
```

## 2. 系統監控與日誌管理

MHH 系統日誌包含應用程式日誌（檔案）與業務稽核日誌（資料庫）。

### 2.1 應用程式日誌 (D:\MHH\logs)
| 檔名 | 說明 | 維運重點 |
|:--|:--|:--|
| `sys.log` | Web 應用 (mhh-ap) 的執行日誌 | 監控 `ERROR` 關鍵字，特別是資料庫斷線或 SSO 失敗。 |
| `batch.log` | 排程任務 (mhh-batch) 的執行日誌 | 檢視 Job 執行時間與解析 PDF 時的異常。 |

### 2.2 業務稽核日誌 (DB)
- **資料表**: `USER_LOGS`
- **維護**: 每 1 年執行一次 `LogCleanupJob` 自動清理。

## 3. 資料庫監控 (SQL Server)

維運人員可使用以下 SQL 快速檢查系統狀態：

### 3.1 檢查排程任務狀態
```sql
-- 檢查各類 Job 的最後執行時間與啟用狀態
SELECT JOB_NAME, CRON_EXPRESSION, IS_ENABLED, LAST_RUN_TIME 
FROM JOBS_CONF;
```

### 3.2 檢查電文同步進度 (SWAL)
```sql
-- 檢查最近一小時內從 SWAL 同步進來的電文數量
SELECT COUNT(*) FROM MSG_INCOMING 
WHERE SYNC_TIME > DATEADD(HOUR, -1, GETDATE());
```

### 3.3 檢查 PDF 解析失敗紀錄
```sql
-- 若 PdfImportJob 遇到無法解析的 PDF 會記錄於此
SELECT * FROM SYS_LOGS 
WHERE LOG_TYPE = 'PDF_ERROR' ORDER BY LOG_TIME DESC;
```

## 4. 故障排除 (Troubleshooting)

| 問題現象 | 可能原因 | 處理建議 |
|:--|:--|:--|
| **無法開啟 Web 畫面** | Java 服務未啟動或 Port 被佔用 | 使用 `netstat -ano | findstr 8080` 檢查；重啟 `mhh-ap.jar`。 |
| **電文遲遲未更新** | `mhh-batch` 服務當機或排程被停用 | 檢查 `JOBS_CONF` 是否啟用；重啟 `mhh-batch.jar`。 |
| **PDF 解析失敗** | PDF 格式非標準或受損 | 檢查 `D:\MHH\data\ERROR` 目錄，手動檢視報錯 PDF。 |
| **資料庫連線超時** | DB 負載過高或網路不通 | 嘗試使用 SQL Management Studio 連線測試；檢查防火牆。 |

---

## 5. 定期維護工作

1. **磁碟空間檢查**：定期清理 `D:\MHH\data\ARCHIVE` 中超過保存年限（如 7 年）的 PDF。
2. **密碼更換**：若更換資料庫密碼，須同步更新各環境的 `application.yml` 並重啟服務。
3. **性能優化**：定期重建 `MSG_INCOMING` 與 `MSG_OUTGOING` 的常用索引。
