## Part A：DEV 開發環境建置

本節說明開發人員於本地環境之開發與測試流程。

### A.1 前置需求
- **JDK**: 21 (推薦使用 OpenJDK 或 Temurin)
- **Maven**: 3.9+
- **Docker Desktop**: 用於啟動本地 MS SQL Server 環境
- **IDE**: IntelliJ IDEA 或 VS Code (推薦安裝 Java 擴充套件)

### A.2 本地程式啟動步驟 (DEV Mode)

1. **啟動資料庫**：
   進入 `DB-init` 目錄並透過 Docker 啟動，系統將自動執行 Schema 初始化：
   ```powershell
   cd DB-init
   docker-compose up -d
   ```

2. **全模組編譯**：
   在專案根目錄執行：
   ```powershell
   mvn clean install -DskipTests
   ```

3. **啟動 Web 服務 (mhh-ap)**：
   ```powershell
   mvn spring-boot:run -pl mhh-ap
   # 預設 Port: 8080
   # DEV 模式登入: admin / 1234 (由 mhh.auth.dev-mode: true 控制)
   ```

4. **啟動排程服務 (mhh-batch)**：
   ```powershell
   mvn spring-boot:run -pl mhh-batch
   # 用於測試 PDF 解析與 SWAL 同步功能
   ```

## 常見問題處理 (DEV/PROD 通用)

- **連線失敗**：檢查 `application.yml` 中的 `datasource` 設定與資料庫防火牆。
- **PDF 不會自動處理**：確認 `mhh-batch` 是否已啟動，且 `JOBS_CONF` 資料表中的 `PdfImportJob` 為啟用狀態 (`IS_ENABLED=1`)。
