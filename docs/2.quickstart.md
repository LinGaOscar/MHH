# 2. 快速啟動指南 (Quickstart Guide)

本文件提供開發人員（DEV）與維運人員（PROD）快速部署與啟動 MHH 系統的步驟。

> 建議閱讀順序：[1. 系統概覽](1.overview.md) → **本文件**

---

## Part A：DEV 開發環境建置

本章節適用於開發人員在本地端進行開發與測試。

### A.1 前置需求
- **JDK**: 21 (建議使用 OpenJDK 或 Temurin)
- **Maven**: 3.9+
- **Docker Desktop**: 用於本地快速啟動 MS SQL Server
- **IDE**: IntelliJ IDEA 或 VS Code (推薦安裝 Java 擴充套件)

### A.2 本地程式啟動步驟 (DEV Mode)

1. **啟動資料庫**：
   進入 `DB-init` 目錄並使用 Docker 啟動，系統會自動初始化 Schema：
   ```powershell
   cd DB-init
   docker-compose up -d
   ```

2. **全模組編譯**：
   在專案根目錄執行：
   ```powershell
   mvn clean install -DskipTests
   ```

3. **啟動 Web 服務 (mhh-ap)**：
   ```powershell
   mvn spring-boot:run -pl mhh-ap
   # 預設 Port: 8080
   # DEV 模式登入: admin / 1234 (由 mhh.auth.dev-mode: true 控制)
   ```

4. **啟動排程服務 (mhh-batch)**：
   ```powershell
   mvn spring-boot:run -pl mhh-batch
   # 用於測試 PDF 解析與 SWAL 同步功能
   ```

## Part B：PROD 生產環境部署規範

本章節提供維運人員將系統部署至生產環境 Windows Server 的標準流程。

### B.1 環境準備
- **Runtime**: Java 21 JRE
- **Database**: 獨立安裝之 MS SQL Server 2019+
- **目錄權限**: 執行帳戶須對電文目錄（如 `D:\MHH_FILES`）具有完全控制權限。

### B.2 打包與部署流程

1. **執行打包**：
   在開發環境或 CI 伺服器執行：
   ```powershell
   mvn clean package -DskipTests
   ```

2. **收集產出物**：
   - 網頁應用：`mhh-ap/target/mhh-ap.jar`
   - 排程任務：`mhh-batch/target/mhh-batch.jar`
   - 配置文件：從 `src/main/resources/application.yml` 複製並依環境修改。

3. **PROD 啟動步驟**：
   建議將 JAR 與 `application.yml` 分開存放，使用以下指令啟動：

   **啟動 Web 應用：**
   ```powershell
   java -jar mhh-ap.jar --spring.config.location=file:./config/application-prod.yml
   ```

   **啟動 Batch 服務：**
   ```powershell
   java -jar mhh-batch.jar --spring.config.location=file:./config/application-prod.yml
   ```

### B.3 目錄結構規範 (建議)
```text
D:\MHH\
├── bin\           # 存放 mhh-ap.jar, mhh-batch.jar
├── config\        # 存放 application-prod.yml
├── logs\          # 系統產生的日誌檔
└── data\          # 電文處理目錄 (PDF 來源、歸檔、暫存)
```

## 常見問題處理 (DEV/PROD 通用)

- **連線失敗**：檢查 `application.yml` 中的 `datasource` 設定與資料庫防火牆。
- **PDF 不會自動處理**：確認 `mhh-batch` 是否已啟動，且 `JOBS_CONF` 資料表中的 `PdfImportJob` 為啟用狀態 (`IS_ENABLED=1`)。
