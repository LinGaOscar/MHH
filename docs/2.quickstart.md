# 2. 快速啟動指南 (Quickstart Guide)

本文件提供開發人員（DEV）與維運人員（PROD）快速部署與啟動 MHH 系統的步驟。

> 建議閱讀順序：[1. 系統概覽](1.overview.md) → **本文件**

---

## Part A：DEV 開發環境建置

本章節適用於開發人員在本地端進行開發與測試。

### A.1 前置需求
- **JDK**: 21 (建議使用 OpenJDK 或 Temurin)
- **Maven**: 3.9+
- **Docker Desktop**: 用於本地快速啟動 MS SQL Server
- **IDE**: IntelliJ IDEA 或 VS Code (推薦安裝 Java 擴充套件)

### A.2 本地程式啟動步驟 (DEV Mode)

1. **啟動資料庫**：
   進入 `DB-init` 目錄並使用 Docker 啟動，系統會自動初始化 Schema：
   ```powershell
   cd DB-init
   docker-compose up -d
   ```

2. **全模組編譯**：
   在專案根目錄執行：
   ```powershell
   mvn clean install -DskipTests
   ```

3. **啟動 Web 服務 (mhh-ap)**：
   ```powershell
   mvn spring-boot:run -pl mhh-ap
   # 預設 Port: 8080
   # DEV 模式登入: admin / 1234 (由 mhh.auth.dev-mode: true 控制)
   ```

4. **啟動排程服務 (mhh-batch)**：
   ```powershell
   mvn spring-boot:run -pl mhh-batch
   # 用於測試 PDF 解析與 SWAL 同步功能
   ```

## 常見問題處理 (DEV/PROD 通用)

- **連線失敗**：檢查 `application.yml` 中的 `datasource` 設定與資料庫防火牆。
- **PDF 不會自動處理**：確認 `mhh-batch` 是否已啟動，且 `JOBS_CONF` 資料表中的 `PdfImportJob` 為啟用狀態 (`IS_ENABLED=1`)。
