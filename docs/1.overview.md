# 1. 系統全局概覽 (System Overview)

本文件提供 MHH (Message History Hub) 系統的架構圖與核心設計理念，供維運與開發人員快速掌握系統全貌。

> 建議閱讀順序：**本文件** → [2. 快速啟動](2.quickstart.md) → [4.1 技術設計導覽](4.design_guide.md)

---

## 1. 核心目標

| 目標 | 說明 |
|:--|:--|
| **集中存儲** | 將來自不同管道（SWAL 資料庫、原始 PDF）的電文整合至 MHH 資料庫 |
| **自動化處理** | 透過排程服務減少人工干預，支援動態 Cron 配置 |
| **多樣化支援** | 採用策略模式，同步支援 ISO 20022 MX 與 SWIFT MT 電文解析 |
| **稽核合規** | 完整記錄使用者行為（查詢、下載、審核），符合金融稽核標準 |

## 3. 核心數據流

### 3.1 SWAL 資料庫同步 (進/出電)
- **來源**: 外部 Oracle SWAL 資料庫。
- **機制**: `SwalSyncJob` 透過增量標記 (Watermark) 每批同步 500 筆電文。
- **目的**: 建立可供 UI 快速檢索的索引資料。

### 3.2 PDF 電文匯入
- **來源**: 特定檔案目錄 (`/data/MX`, `/data/MT`)。
- **機制**: `PdfImportJob` 掃描目錄並透過 `ParserFactory` 識別電文類型進行解析。
- **目的**: 支援從原始報表或檔案系統自動匯入電文。

### 3.3 預約下載流程
1. 使用者送出下載申請 (Maker)。
2. 主管於系統放行 (Checker)。
3. `ReservationMergeJob` 背景自動打包 PDF 並存於 `/data/TEMP`。

## 4. 模組職責說明

| 模組 | 職責 | 核心技術 |
|:--|:--|:--|
| `mhh-common` | 定義 JPA 實體與共用工具類 | Spring Data JPA, Hibernate |
| `mhh-core` | 解析器框架與電文處理邏輯 | Java Strategy Pattern |
| `mhh-ap` | Web 頁面展示與 API 服務 | Spring Boot, Spring Security |
| `mhh-batch` | 執行所有背景排程任務 | Spring TaskScheduler |
